# 산업수학I 기말 프로젝트
카페 주문을 관리하는 간단한 웹 애플리케이션

## 카페 주문 시스템

```
* 회원가입 및 로그인: 사용자는 이메일과 비밀번호를 사용하여 회원가입하고 로그인할 수 있음
* 메뉴 관리: 관리자는 메뉴 항목을 추가하고 삭제할 수 있음
* 주문 처리: 사용자는 주문을 생성하고, 관리자는 주문을 확인하고 처리할 수 있음
```

```
1. signup
    회원가입 처리: 서버에 회원가입 요청을 보내기 전에 이메일, 비밀번호 및 비밀번호 확인 폼이 조건에 충족되었는지 확인
    서버와 통신: fetch API를 사용해 서버에 POST 요청을 보냄
    응답 처리: 서버로부터 받은 응답 확인 후 정상적으로 처리되지 않으면 오류처리 후 오류 메시지 표시

2. login, logout
    로그인 처리: 입력된 이메일과 비밀번호 정보를 가져와 서버에 로그인 요청을 보냄
    서버와 통신: fetch API를 사용해 서버에 POST 요청을 보냄
    응답 처리: 서버로부터 받은 응답이 정상적으로 처리되면 세션 스토리지에 사용자 이메일을 저장하고 쿠키를 설정한 후 관리자 페이지로 리다이렉트함
    로그아웃 처리: 서버로부터 받은 응답이 정상적으로 처리되면 세션 스토리지를 비우고 메인 페이지로 리다이렉트함

3. menu
    주문 처리: 사용자가 메뉴 항목을 클릭하면 해당 메뉴를 장바구니에 추가하고, 클릭한 메뉴를 장바구니에 추가하고 화면에 표시함 
    주문 버튼을 클릭하면 장바구니에 항목이 있는지 확인하고, 비어있지 않으면 서버에 주문을 보냄

4. admin
    메뉴 항목 추가 및 삭제: 서버에서 가져온 메뉴 항목을 화면에 표시하고, 사용자가 메뉴를 클릭하여 삭제할 수 있는 기능을 구현함
    메뉴이름, 가격, 이미지 파일을 입력받아 메뉴에 추가하는 기능을 구현함

5. db
    자동 로드 및 자동 저장: LokiJS는 자동으로 데이터베이스를 로드하고 저장할 수 있는 기능을 제공함
    간단한 API: LokiJS는 간단하고 직관적인 API를 제공하여 데이터베이스에 데이터를 추가, 조회, 수정, 삭제할 수 있음

6. index
    파일 시스템 및 디렉토리 생성: uploads 디렉토리를 만들고, 이미 존재하지 않는 경우에만 생성
    Express 미들웨어 설정: express.json(), express.static(), cookieParser() 등의 미들웨어를 설정하여 요청의 JSON 파싱, 정적 파일 제공, 쿠키 파싱 등을 처리
    클라이언트 요청 처리: 엔드포인트를 통해 사용자의 로그인, 로그아웃, 회원가입 등을 처리
    서버 실행: Express 애플리케이션을 3000번 포트에서 실행
```

## 피드백
```
* 쿠키를 이용한 로그인 상태 확인 구현 필요
* 주문처리를 어떻게 할 것인지, 결제창으로 넘어가는 등의 다음단계 구현 필요
* 장바구니 주문 취소버튼 가운데 정렬 CSS 오류 확인 후 수정 필요
```

### 코드 틀
```
* Express.js: Node.js 웹 애플리케이션을 위한 웹 프레임워크로, 서버 측 라우팅 및 미들웨어를 구축하는 데 사용됨
* LokiJS: 자바스크립트로 작성된 내장형 NoSQL 데이터베이스로, 메모리 내 데이터 저장 및 쿼리를 처리하는 데 사용됨
* Formidable: 파일 업로드 및 폼 데이터 처리를 위한 Node.js 모듈로, 이미지 업로드 및 멀티파트 폼 데이터를 처리하는 데 사용됨
* Fetch API: 웹 브라우저에서 제공되는 비동기 네트워크 요청을 처리하는 인터페이스로, 클라이언트와 서버 간의 데이터 통신에 사용됨
* File System (fs): Node.js의 내장 모듈 중 하나로, 파일 및 디렉토리 생성, 삭제, 수정 등 파일 시스템 작업에 사용됨
* Path (path): Node.js의 내장 모듈 중 하나로, 파일 및 디렉토리 경로를 조작하는 데 사용됨
* Cookie Parser: Express 미들웨어로, 요청에서 쿠키를 구문 분석하여 쿠키를 파싱하고 쿠키 기반의 세션 관리에 사용됨
```
